# Cline共創開発ログ：ダーツ練習数値入力機能の実装

このドキュメントは、AI（Cline）と人間が対話を通じて、どのように要件定義、実装、そしてトラブルシューティングを進めたかを詳細に記録したものです。

## 1. 開発概要
ダーツ練習支援システム「Naiguru Insights」に、従来の自由記述による振り返りに加え、BULL数やプレイ時間などの数値をLINEから対話形式で入力できる機能を追加した。

*   **対応バージョン**: v1.6.0 〜 v1.6.1
*   **主な成果**: 対話型数値入力、hh:mm形式サポート、連投耐性（排他制御）

## 2. 開発プロセス詳細

### フェーズ1：要件の具体化
**【プロンプト】**
> 'Scripts/config.gs' のCOLに記載の BULL_COUNT, LOWTON_COUNT 等について、LINEの画面から入力できるようにしたい。

**【Clineのアクション】**
*   `config.gs` を読み取り、将来の項目追加を容易にするため、配列による動的定義を提案。
*   当初は「特定のキーワード（B:11など）での一括入力」を提案したが、ユーザーより「一項目ずつ順番に聞く対話形式」へのブラッシュアップ依頼を受けた。

**【対話の変遷】**
*   ユーザー: 「ひとつずつ順番に聞いて、ループすることはできないか？」
*   Cline: シートの空欄を判定して「次に聞くべき項目」を動的に判断するロジックを提案。

### フェーズ2：不具合発生と原因分析
初期実装後、数値が期待した列（L列など）ではなく、振り返り文章の列（F列）を上書きしてしまう不具合が発生した。

**【トラブルシューティングの流れ】**
1.  **Clineの初期分析**: `SpreadsheetApp.flush()` 不足によるキャッシュの影響と推測。
2.  **不具合の継続**: `flush()` を入れても上書きが止まらない。
3.  **ユーザーの指摘**: 「振り返り入力中の特定列にピンポイントで入力しに行けないか？」という本質的な問いかけ。
4.  **Clineの再分析**: 
    *   `REVIEW_READY` ステータス内で振り返りと数値を両方判定していたため、ロジックが混同していたことを特定。
    *   新ステータス `DATA_INPUT` を導入し、保存先を完全に分離することを決断。

### フェーズ3：連投（競合）問題の特定と解決
テスト中に再び上書きが発生。ここで「実行環境特有の挙動」が浮き彫りになった。

**【プロンプト】**
> 現在テストとして、短いスパンで入力しているのは関係ありますか？

**【Clineのアクション】**
*   **「競合状態（Race Condition）」** を特定。LINE Webhookが並列で実行される際、ステータス更新が間に合わず古いステータスを参照してしまうのが真因。
*   **排他制御（LockService）** の導入を提案。

**【解決策の説明】**
Clineは「信号機の設置」という例えを用い、処理を順番待ちにすることで整合性を保つ仕組みを説明し、`lock.waitLock(10000)` を実装。

### フェーズ4：追加要件の即時反映
**【プロンプト】**
> DARTSLIVEプレイ時間ですが、hh:mm形式をとります。入力できるよう修正お願いします。

**【Clineのアクション】**
*   バリデーションロジックを拡張。`config.gs` に `type: 'time'` 定義を追加し、正規表現による形式チェックを実装。
*   質問メッセージに自動で「（例：1:30）」と例示が出るように親切なUIへ改善。

## 3. 技術的な知見（Wiki向け）
今回の開発で得られた、Clineを使いこなすためのヒント。

1.  **不具合の伝え方**: ユーザーが「どこに、何が、どのように誤って入っているか」を具体的に伝えたことで、 Clineはソースコードとシート構造を正しく突き合わせることができた。
2.  **ステータス管理**: シンプルさを追求しすぎると、かえってロジックが複雑になりバグを産む。明示的なステータス（`DATA_INPUT`）の導入が、結局は最短の解決策となった。
3.  **並列処理の考慮**: GAS×LINEのような非同期システムでは、`LockService` による排他制御が必須であることをClineが提示した。

*   `Documents/add_new_metric.md`: 将来の拡張手順。
*   `Documents/line_flow_guide.md`: LINE上の対話フロー図。
